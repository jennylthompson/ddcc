This page provides a list of the FHIR artifacts defined as part of this implementation guide.

Behavior: Capability Statements

The following artifacts define the specific capabilities that different types of systems are expected to have in order to comply with this implementation guide. Systems conforming to this implementation guide are expected to declare conformance to one or more of the following capability statements.

WHO SHC Registry Service

CapabilityStatement for WHO SHC Registry Service actor.
WHO SHC Generator

CapabilityStatement for WHO SHC Generator actor.

Structures: Graph Definitions

These define collections of interrelated resources that are useful to pass around as a group

GraphDefinition/svc-graph

SHF Graph Definition for retrieving Smart Vaccination Certificates under a Retrieve Smart Health Folder transaction.

Structures: Questionnaires

These define forms used by systems conforming to this implementation guide to capture or expose data to end users

SVC Questionnaire

SVC QR Code Questionnaire for an Immunization event.

Structures: Abstract Profiles

These are profiles on resources or data types that describe patterns used by other profiles, but cannot be instantiated directly. I.e. instances can conform to profiles based on these abstract profiles, but do not declare conformance to the abstract profiles themselves.

SHC Bundle

An SHC Bundle is a document bundle containing the Smart Health Certificate (SHCs).
SHC Questionniare

​Describes the logical structure for a questionnaire associacted to a Smart Health Certificate which includes a Structure Map to generate the relevant SHC from a given SHC QuestionnaireResponse which responds to the SHC Questionnaire.
SHF Graph Defintion

Abstract Graph Definition for Smart Health Folder to describe information retrieved under a Retrieve Smart Health Folder transaction.
SHC Questionnaire Response

Describes the logical structure for a questionnaire response associacted to a SHC Questionaire.
SHC Composition

Describes the logical structure for a Smart Health Certificate.

Structures: Resource Profiles

These define constraints on FHIR resources for systems conforming to this implementation guide

Patient (IPS)

This profile represents the constraints applied to the Patient resource by the International Patient Summary (IPS) FHIR Implementation Guide and describes the minimum expectations for the Patient resource when used in the IPS composition or in one of the referred resources.
Immunization (IPS)

This profile represents the constraints applied to the Immunization resource by the International Patient Summary (IPS) FHIR Implementation Guide. A record of an immunization is represented in the patient summary as an instance of an Immunization resource constrained by this profile. It describes the event of a patient being administered a vaccination or a record of a vaccination as reported by a patient, a clinician or another party.
SVC Questionnaire Response

​Describes the logical structure for a questionnaire response associacted to a SVC Questionaire.
SHC Practitioner

Container for an SHC Practitioner, which is a Practitioner authorized by the PHA to vaccinate an SHC Patient.
SHC Patient

An SHC_Patient should:

  • ​if the [identifier.assigner](StructureDefinition-shc-patient-definitions.html#Patient.identifier] is set then the value of identifier.assigner should be set to reference the SHC Organization resource associated to the PHA via the WHO Public Key Directory
  • obeys who-shc-patient-1
SHC Submit Health Event Response

​An SHC Submit Health Event Response is a transaction reponse bundle containing the operation outcomes for the submission of one or more health certificates under the Submit Health Event transaction.
SHC Submit Health Event Request

An SHC Submit Health Event Request is a transaction bundle containing the content of health events which are being submitted under the Submit Health Event transaction.
SHC Organization

​Abstraction for an SHC Organization. There should be an SHC Organization associated to each public health authortiy (PHA) in the WHO Public Key Directory
Provenance of SHC

Used to assert the provenance of a public health authority (PHA) over either a Paper or a Digital Smart Vaccination Certificate (SHC).
SHC Practitioner Role

​Association of an SHC Practitioner to a managing SHC Organization, indicating that an SHC Practitioner is authorized by the public health authority (PHA) to vaccinate an SHC Patient.
SHF Composition

Describes the logical structure for the Smart Health Folder which contains the Smart Health Certificates (SHCs). An SHF Composition should:

  • ​be accessible from a SHC Registry Service as located via the WHO Public Key Directory for the PHA referenced in the author.
SVC Immunization

An SVC Immunization contains the content corresponding to a row in the Vaccination Events section of a Paper SVC. While this is not a further profile of the International Patient Summary (IPS) Immunization resouce, this is intended to map into an IPS Immunization resource.
SVC Bundle

An SVC Bundle is a document bundle containing the Smart Vaccination Certificate (SVC) content.
SVC Basic QR Code resource

SVC Profile of Basic resource for QR Code.
SVC Composition

Describes the logical structure for the health content contained in a Smart Vaccination Certificate (SVC).
